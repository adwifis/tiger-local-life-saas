# 架构说明

## 技术栈

- Web：Next.js + React + TypeScript
- 数据层：Prisma + PostgreSQL
- 缓存：Redis
- AI：阿里云百炼 OpenAI 兼容接口
- 部署：Docker + 阿里云服务器

## 当前阶段架构策略

当前不是从零重写，而是基于 `tiger_ai_saas` 参考项目的通用骨架做二次开发。

保留：

- Next.js 运行骨架
- Prisma / Postgres
- Dockerfile / docker-compose
- 环境校验脚本
- 百炼快捷命令

新增：

- 商家、代理、平台三套后台信息架构
- 门店资料和模板库
- 内容生成与额度系统
- 代理客户归属

## 业务主链路

1. 商家创建门店资料
2. 选择营销模板
3. 提交生成参数
4. 百炼返回结构化内容
5. 保存生成记录并扣减额度
6. 代理可切换客户门店继续生成
7. 平台后台开通套餐和管理模板

## 开发原则

1. 先保证商用闭环，再扩功能。
2. 先手工开通套餐，再接自动支付。
3. 先角色隔离和日志，再做高级自动化。
4. 先单体项目快速上线，不提前微服务化。
